# Security Policy

## Supported versions

Until ONX has a formal maintenance policy, security fixes are applied to the latest main branch state first.

## Reporting a vulnerability

Please do **not** open a public GitHub issue for a suspected security problem.

Instead, report it privately to the maintainer before disclosure. Include:

- affected version or commit
- reproduction steps
- expected impact
- any suggested mitigation

If a dedicated security contact is added later, this file should be updated to point to that address or intake channel.
